# Implementation Plan: Companies (Accounts) Module

**Branch**: `feat/companies-module` | **Date**: 2026-06-14 | **Spec**: [spec.md](spec.md)

**Input**: Feature specification from `agent-os/specs/companies/spec.md`

---

## Summary

Build the Companies module — the first CRM entity — providing full CRUD via Django REST Framework on the backend and a Next.js 16 / TanStack Query frontend with list, detail, create, and edit pages. The backend uses the existing `TimestampedModel` + `SoftDeleteMixin` base classes, JWT Bearer auth, and a custom `PageNumberPagination` subclass to satisfy the spec's 1-based `page` / `page_size` URL contract. All four frontend pages encode their search/filter/sort/page state in the URL. The companies app has no import-time dependency on contacts, leads, or deals.

---

## Technical Context

**Language/Version**: Python 3.12 (backend) · TypeScript 5 (frontend)

**Primary Dependencies**:
- Backend: Django 5.2, DRF, drf-spectacular, django-filters, adrf (async views), simplejwt
- Frontend: Next.js 16, React 19, TanStack Query v5, TanStack Table v8, React Hook Form 7, Zod 4, Tailwind CSS v4

**Storage**: MySQL (utf8mb4) via `DATABASE_URL` / django-environ

**Testing**: pytest + pytest-django + factory_boy (backend) · Vitest + Testing Library (frontend unit) · Playwright (frontend e2e)

**Target Platform**: Web application — Django REST API + Next.js frontend, deployed separately

**Project Type**: Web application (fullstack, separate backend/frontend)

**Performance Goals**: Company search results within 2 s on standard broadband (SC-001); list updates without full page reload (SC-002)

**Constraints**:
- Pagination max `page_size` = 100 (FR-008, edge-cases section)
- JWT Bearer auth on every endpoint; unauthenticated → 401 (FR-017)
- Soft delete only — no hard deletes (FR-013)
- No import-time or runtime dependency on contacts/leads/deals (FR-018)

**Scale/Scope**: Single CRM team, phase 1 — no record-level permission scoping
